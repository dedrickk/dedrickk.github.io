# Product

## Register

brand

## Users

Recruiters, hiring managers, and fellow engineers evaluating Dedrick D. for DevOps / Cloud / SRE roles. They arrive skeptical, skim fast, and decide in seconds whether this person ships production-grade work. Many have seen a hundred identical template portfolios the same day.

## Product Purpose

A personal portfolio that *proves* DevOps craft rather than asserting it — and lands interviews. The design itself is the first artifact in the portfolio: if the site is precise, fast, and uncompromising, it is evidence the engineer is too. Success = a visitor reaches the project case studies, understands the engineering depth (observability, CI/CD, IaC), and reaches out.

## Brand Personality

Precise. Intense. Uncompromising. The governing line is **"Mediocrity is a sin."** The voice is a disciplined operator at a terminal at night — confident, exacting, a little dangerous — not a cheerful SaaS marketing page. Edge carried by a committed crimson identity (continuous with the owner's GitHub README and the Berserk *Brand of Sacrifice* mark), restraint carried by typography and exacting execution.

## Anti-references

- Generic AI-SaaS portfolios: blue→cyan gradients, gradient text, glassmorphism cards, animated radial-blob backgrounds. (This is exactly what the site looked like before — the redesign is a deliberate repudiation of it.)
- Linktree / template-marketplace portfolios with identical icon-heading-text card grids.
- Cream / editorial-magazine aesthetic (display-serif + italic + drop caps). Wrong register entirely.

## Design Principles

1. **Practice what you preach.** The site is itself production-grade — accessible, fast, responsive, bug-free. A DevOps portfolio that ships sloppy front-end undercuts its own claim.
2. **Show the work, not adjectives.** Real (sanitized) architecture, real metrics, real engineering decisions. No "passionate, results-driven" filler.
3. **Intensity with discipline.** Bold crimson and a sharp dark world, executed with restraint — committed, never chaotic.
4. **One identity, end to end.** The README, the GitHub profile, and this site read as the same person. Continuity is the brand.

## Accessibility & Inclusion

WCAG 2.1 AA. Body text ≥4.5:1 on the ink background; large text/headings ≥3:1 (crimson reserved for large/accent use, never small body). Full `prefers-reduced-motion` support (reveals collapse to instant/crossfade). Keyboard-navigable with visible focus rings. Dark-only by deliberate scene choice.
